- 内容来源：`PRDs/网页简历 PRD.md`
- 教程参考：https://wikimind.top/tutorial/ai-web-resume
- 新视觉参考图：`q.png`、`w.png`、`e.png`、`r.png`、`t.png`、`y.png`、`u.png`、`i.png`

本版设计以新参考图为准，放弃上一版浅色、温和作品集方向，改为更有个人站辨识度的视觉风格：深色叙事首页、顶部悬浮胶囊导航、超大粗体标题、绿色到深蓝渐变、倾斜卡片、照片/项目素材拼贴、黑白强反差页面切换。

# 一、视觉风格规范

## 1. 整体视觉调性

### 关键词

- 深色
- 高反差
- 叙事感
- 技术感
- 游戏感
- 年轻
- 个人作品站
- 有一点酷，但不浮夸

### 设计方向

网页不做传统简历，也不做普通卡片堆叠页面。整体应像一个“个人成长档案 + 技术作品站”：第一屏用强视觉记忆点吸引访问者，后续用 Work / Story / Life 三条路径组织内容。

内容基调仍然保持真实：黄志铭是 UTS 人工智能本科大一学生，正在积累 AI / IT / 软件开发 / Minecraft 工具项目。视觉可以大胆，但文字不能夸大经历。

### 页面叙事结构

顶部导航使用三个主入口：

- Work：作品项目、技能、GitHub
- Story：关于我、学习经历、成长路径
- Life：B 站、留学生活、内容输出、联系

页面仍保留 PRD 中的 7 个板块，但视觉上可以按这三个入口分组，而不是做成传统长简历。

## 2. 配色方案

### 主色板

| 用途 | 色值 | 使用说明 |
|---|---:|---|
| 深黑背景 | `#030303` | Story、Hero 下半段、深色卡片背景 |
| 墨黑卡片 | `#181818` | 悬浮导航、倾斜卡片、深色信息卡 |
| 卡片边框 | `#2A2A2A` | 深色卡片描边和分隔线 |
| 主白文字 | `#FFFFFF` | 大标题、重要标题 |
| 次级灰文字 | `#A8A8A8` | 正文、说明文字 |
| 弱提示文字 | `#6F6F6F` | 占位、状态、辅助信息 |
| 浅色页面背景 | `#F7F7F5` | Work 白底板块 |
| 浅色边框 | `#E2E2DF` | 白底卡片描边 |

### 渐变与强调色

| 用途 | 色值 | 使用说明 |
|---|---:|---|
| Hero 浅绿 | `#C8FFD7` | 第一屏顶部渐变起点 |
| Hero 薄荷绿 | `#5EE0A1` | 第一屏中段，表达年轻与学习成长 |
| Hero 青绿 | `#1D9A8A` | 第一屏下沉过渡 |
| Hero 深蓝 | `#06233F` | 第一屏底部，连接深色叙事区 |
| 活跃绿点 | `#4CFFB0` | 导航 active、状态点、关键小强调 |
| 技术蓝 | `#286DFF` | GitHub、Web 工具、JavaScript 项目 |
| Minecraft 金 | `#F0B13C` | Minecraft 项目局部强调 |
| B 站粉 | `#FB7299` | B 站入口和内容输出 |

### 配色规则

- Hero 第一屏使用大面积绿色到深蓝渐变。
- Story、关于我、学习经历等叙事板块优先使用黑底。
- Work、项目网格、技能和内容平台可切换到白底，形成滚动节奏。
- 强调色只用于状态点、标签、项目关键词和链接，不铺满整块内容。
- 黑底正文必须用 `#A8A8A8` 以上亮度，避免灰得看不清。

## 3. 字体规范

### 字体族

优先使用粗壮、现代的无衬线字体：

- 英文与数字：Inter、Satoshi、Geist、SF Pro Display、Segoe UI
- 中文：Noto Sans SC、PingFang SC、Microsoft YaHei
- 兜底：sans-serif

### 字号层级

| 层级 | 桌面端 | 移动端 | 使用位置 |
|---|---:|---:|---|
| Hero 超大标题 | 96-150px | 52-72px | 第一屏主标题、姓名 |
| 大板块标题 | 64-96px | 40-56px | Work / Story / Life 分区标题 |
| 板块标题 | 42-56px | 30-38px | 关于我、项目、技能、联系 |
| 项目标题 | 24-32px | 22-26px | 项目卡片、经历卡片 |
| 正文 | 20-24px | 16-18px | 自我介绍、项目说明 |
| 辅助文字 | 15-17px | 13-15px | 时间、状态、标签说明 |
| 胶囊导航 | 16-18px | 14-15px | 顶部导航 |

### 字重规则

- 超大标题使用 800-900。
- 板块标题使用 750-850。
- 正文使用 450-500。
- 标签和导航使用 600。
- 不使用负字距，避免中文标题挤压。

## 4. 页面宽度和间距

### 主体宽度

- 桌面端内容最大宽度：`1120px`
- Hero 视觉区域可突破到 `1280px`
- 顶部悬浮导航宽度：`min(1120px, calc(100vw - 32px))`
- 主体左右边距：桌面 48px，移动 20px

### 板块间距

| 场景 | 桌面端 | 移动端 |
|---|---:|---:|
| Hero 高度 | 100vh | 92-100vh |
| 大分区上下留白 | 140-220px | 88-128px |
| 标题到正文 | 28-44px | 20-28px |
| 卡片间距 | 24-32px | 16-20px |
| 导航顶部距离 | 8-16px | 8px |

### 视觉节奏

页面需要明显的“黑 / 白 / 黑 / 白”节奏：

1. Hero：绿色渐变 + 深蓝黑
2. Story：黑底叙事
3. Work：白底项目网格
4. Skills：白底或浅灰
5. Life：黑底或渐变
6. Contact：深色收尾

## 5. 顶部悬浮导航

### 样式

- 固定在页面顶部居中。
- 背景：`rgba(24, 24, 24, 0.92)`。
- 圆角：12-14px。
- 高度：54-62px。
- 边框：`1px solid rgba(255,255,255,0.10)`。
- 模糊：`backdrop-filter: blur(16px)`。
- 阴影：`0 16px 40px rgba(0,0,0,0.28)`。

### 结构

左侧：

- 圆形个人标识：可用 `XM`、绿蓝钻石图标、像素方块图标。
- 导航项：Work、Story、Life。

右侧：

- “Say hi!” 联系入口。
- GitHub 图标。
- B 站或邮箱图标。
- 主题/关闭风格的装饰图标可选。

### active 状态

- 当前栏目文字变白。
- 下方显示一颗小绿点或短线。
- 未激活项使用 `#A8A8A8`。

### 移动端

- 导航保留悬浮样式，但隐藏部分社交图标。
- Work / Story / Life 保留。
- “Say hi!” 可缩短为“Hi”或只保留邮箱图标。

## 6. 圆角规范

| 元素 | 圆角 |
|---|---:|
| 顶部悬浮导航 | 12-14px |
| 深色故事卡片 | 16-22px |
| 白底项目卡片 | 14-18px |
| 小图标按钮 | 10-12px |
| 胶囊标签 | 999px |
| 照片/截图贴纸 | 8-14px |

卡片允许比上一版更大圆角，但不要使用太软的“可爱风”圆角。整体应更接近酷感、技术感和作品站。

## 7. 阴影规范

| 类型 | 使用位置 | 建议效果 |
|---|---|---|
| 导航阴影 | 顶部悬浮导航 | `0 16px 40px rgba(0,0,0,0.28)` |
| 深色卡片阴影 | 倾斜叙事卡片 | `0 30px 80px rgba(0,0,0,0.45)` |
| 白底卡片阴影 | Work 项目卡 hover | `0 20px 60px rgba(0,0,0,0.10)` |
| 贴纸阴影 | 照片、截图、图标贴纸 | `0 18px 40px rgba(0,0,0,0.35)` |

深色区域可以更大胆使用阴影和层叠；白底区域保持克制。

## 8. 卡片样式

### 深色倾斜卡片

用于 Hero 下方视觉入口、Story 拼贴、Life 内容：

- 背景：`#181818`
- 边框：`1px solid #2A2A2A`
- 圆角：18px
- 卡片标题区使用顶部横条或分隔线。
- 允许轻微旋转：`-6deg` 到 `6deg`
- hover 时旋转回正或轻微上浮。

### 白底项目卡片

用于 Work 项目网格：

- 背景：`#F7F7F5` 或 `#FFFFFF`
- 边框：`1px solid #E2E2DF`
- 圆角：16px
- 顶部有项目名栏，底部有项目类型和状态。
- 中间区域放项目截图、3D 图标、命令示例或占位视觉。

### 数据/能力模块

用于技能和学习方向：

- 可采用“列表 + 大圆盘/仪表盘/代码块视觉”的左右布局。
- 左侧列出能力项，右侧放一个抽象视觉组件。
- 不使用百分比吹嘘技能，使用“入门了解 / 基础掌握 / 项目练习中”。

## 9. 图片、拼贴和素材方向

### Hero 视觉

可以使用三张深色卡片拼贴：

- Work：展示 Minecraft / AI / Tool 等立体标签。
- Story：展示 UTS、学习路径、头像或照片占位。
- Life：展示 B 站、留学生活、游戏内容、个人照片占位。

### 项目视觉

当前项目截图未完整补齐时，使用正式占位：

- Minecraft MOD：像素方块、服务器菜单、Java/Fabric 标签。
- 附魔指令生成器：命令行 `/give` 示例、输入输出面板。
- 数据库项目：ERD 节点图、SQL 片段。
- Java 项目：终端窗口或类结构图。
- JavaFX 项目：简化界面窗口。

### 贴纸效果

- 照片/截图可带白色描边或浅灰边框。
- 允许旋转和重叠，但不能遮挡正文。
- 每个拼贴区域最多 3-5 个元素，避免凌乱。

## 10. 动效和交互

### 基础动效

- 页面加载：Hero 大标题淡入并轻微上移。
- 悬浮导航：进入后固定顶部，active 点随滚动移动。
- 卡片 hover：上浮 6px，倾斜角度减少，阴影增强。
- 贴纸 hover：轻微旋转或放大到 1.03。
- 链接 hover：文字变白或强调绿，图标轻微位移。

### 滚动动效

- 黑底 Story 区块使用逐段出现。
- Work 项目卡片按行 stagger 淡入。
- Skills 能力列表可根据 hover 高亮右侧视觉组件。

### 交互限制

- 动效要流畅但不炫技。
- 不使用大范围随机漂浮元素。
- 所有动画支持 `prefers-reduced-motion`，减少动效用户直接显示静态内容。

# 二、7 个板块的具体 UI 设计

## 1. Hero 区：姓名、一句话定位、联系方式

### 设计目标

第一屏要有强记忆点。不要像传统简历一样先列联系方式，而是用超大标题和视觉拼贴表达“我是一个正在 UTS 学 AI、用项目记录技术成长的人”。

### 推荐主标题

可选两种方向：

1. 中文主标题：
   - `黄志铭`
   - `用项目记录我的技术成长`
2. 英文混排主标题：
   - `Let's build`
   - `with AI & code`

副标题使用 PRD 中的一句话定位：

> UTS 人工智能本科生，关注 AI 应用、软件开发与游戏工具开发。

### UI 组件

- 顶部悬浮胶囊导航
- 超大 Hero 标题
- 状态点：`UTS · AI Student · Sydney`
- 一句话定位
- 倾斜 “Say hi!” 联系按钮
- 三张深色入口卡片：Work / Story / Life
- 底部滚动提示或项目标识条

### 布局结构

桌面端：

- 导航固定顶部居中。
- Hero 背景从浅绿渐变到深蓝黑。
- 中心放超大标题，标题可换行。
- 标题下方或右下方叠放一个倾斜 “Say hi!” 卡片按钮。
- 下方放 3 张倾斜入口卡片，分别跳转 Work / Story / Life。

移动端：

- 标题缩小但仍保持大字冲击力。
- 三张入口卡片改为纵向堆叠。
- “Say hi!” 按钮不遮挡标题，放在副标题下方。

### 交互效果

- 点击 Work / Story / Life 卡片平滑滚动到对应分区。
- “Say hi!” 点击滚动到联系 CTA。
- 邮箱按钮复制邮箱。
- GitHub、B 站图标新窗口打开。
- Hero 卡片随鼠标轻微 3D 倾斜，移动端关闭该效果。

## 2. 关于我

### 设计目标

参考黑底 Story 页面，把“关于我”做成一个叙事段落，而不是普通简介卡片。重点是：我是谁、为什么学 AI、目前在做什么、未来想往哪里走。

### UI 组件

- 黑底 Story 分区
- 大标题：`How I got here` 或 `关于我`
- 2-3 段正文
- 右侧照片/学习素材拼贴
- 小型状态标签
- “Remember the path?” 风格的小提示或滚动入口

### 布局结构

桌面端：

- 左侧 46%：大标题 + 正文段落。
- 右侧 48%：倾斜照片/素材拼贴。
- 拼贴素材建议：UTS、代码截图、Minecraft 方块、AI/数据库图标、个人照片占位。

移动端：

- 标题和正文在上。
- 拼贴在下，缩小并减少元素数量。

### 交互效果

- 正文段落进入视口时逐段淡入。
- 拼贴元素 hover 时轻微浮起。
- 标签 hover 时显示对应说明，例如 Java、Python、SQL、Minecraft Fabric。

### 内容适配

关于我正文使用 PRD 的 2-3 段内容。不要改写成成熟从业者语气，也不要虚构实习、社团或工作经历。

## 3. 作品和项目

### 设计目标

项目区采用参考图中的白底 Work 页面：大标题、两列项目卡片、卡片中间放大视觉对象。项目卡看起来像“我做过/正在做的作品”，而不是简历表格。

### UI 组件

- 白底 Work 分区
- 大标题：`Things I'm building`
- 项目网格卡片
- 项目状态标签
- 技术栈标签
- 项目视觉占位
- 外链按钮
- 锁定/待补充状态图标

### 布局结构

桌面端：

- 顶部：大标题 + 简短说明。
- 第一行：两个重点项目大卡片。
  - Minecraft 好友服控制菜单 MOD
  - Minecraft 附魔指令生成器
- 第二行：三个课程项目中等卡片。
  - 健康管理应用数据库设计项目
  - Java 商店管理系统
  - JavaFX 图形界面商店系统

卡片结构：

- 顶部栏：项目名 + 状态。
- 中部：大视觉对象或截图占位。
- 底部：项目类型 / 技术栈 / 简短描述。

移动端：

- 项目卡片单列。
- 重点项目仍保留较大视觉区域。
- 课程项目卡片高度降低。

### 交互效果

- 卡片 hover 上浮并增强阴影。
- 项目视觉轻微放大。
- 有链接的项目显示外链箭头；未补充链接显示锁图标或 “Coming soon”。
- 点击项目卡片可展开更多信息：项目简介、亮点、待补充项。

### 项目视觉建议

#### Minecraft 好友服控制菜单 MOD

- 卡片色彩：深黑 + Minecraft 金局部强调。
- 视觉对象：方块风服务器控制面板、菜单窗口、传送点图标。
- 标签：Java、Fabric、GUI、Server Tool。

#### Minecraft 附魔指令生成器

- 卡片色彩：白底 + 技术蓝。
- 视觉对象：命令生成器窗口、`/give` 命令代码片段。
- 标签：HTML、CSS、JavaScript、Minecraft Command。

#### 健康管理应用数据库设计项目

- 视觉对象：ERD 节点图或数据库表关系图。
- 标签：ERD、SQL、Database Design。

#### Java 商店管理系统

- 视觉对象：终端窗口、Product / Store / CashRegister 类结构。
- 标签：Java、OOP、Console App。

#### JavaFX 图形界面商店系统

- 视觉对象：桌面窗口、FXML、MVC 结构。
- 标签：JavaFX、FXML、MVC。

## 4. 经历和履历

### 设计目标

经历区用 Story 的叙事方式呈现学习路径，不伪造成工作履历。重点是“2025.06 进入 UTS，开始人工智能本科和计算机基础学习”。

### UI 组件

- 黑底时间线分区
- 大标题：`My study path`
- 时间线节点
- 学历信息块
- 学习方向标签
- 暂无经历提示

### 布局结构

桌面端：

- 左侧：大标题和说明。
- 右侧：时间线。
- 主要节点为 `2025.06 - 至今`。
- 下方用标签列出 AI 基础、Java、Python、SQL、数据库设计、离散数学、计算机网络等。

移动端：

- 时间线改为竖向单列。
- 节点和标签分组展示。

### 交互效果

- 时间线节点进入视口时点亮绿色状态点。
- 点击学习方向标签可跳到技能区对应分组。
- “暂无实习/社团/志愿者经历”用灰色说明呈现，不做醒目警告。

### 内容规则

必须明确：

- 当前暂无正式实习、兼职、社团或志愿者经历。
- 后续有真实经历后再加入时间线。

## 5. 技能和背书

### 设计目标

参考“能力列表 + 大型圆盘/仪表盘”的图，技能区不做百分比进度条，而做成“我正在训练哪些能力”的模块。

### UI 组件

- 白底能力分区
- 大标题：`How I learn and build`
- 左侧技能列表
- 右侧抽象仪表盘或技能雷达视觉
- 技能标签
- 背书占位卡片

### 布局结构

桌面端：

- 顶部：大标题。
- 中部两列：
  - 左侧：技能分组列表。
  - 右侧：一个抽象圆形仪表盘，4 个刻度对应编程、数据库、工具平台、通用能力。
- 底部：推荐评价 / 背书占位。

移动端：

- 仪表盘放在标题下或隐藏为简化图形。
- 技能列表单列展开。

### 技能分组

- 编程与开发：Java、Python、JavaScript、HTML / CSS。
- 数据库与建模：SQL、ERD、业务规则分析、关系型数据库基础。

## 6. 内容输出

### 设计目标

内容输出区放在 Life 路径中，体现个人主页不仅是简历，也是后续长期更新的内容入口。已有平台重点展示，暂无平台保留占位。

### UI 组件

- Life 黑底或深色渐变分区
- 平台入口卡片
- 平台图标
- 内容方向标签
- 外链按钮
- Coming soon / 待补充状态

### 布局结构

桌面端：

- 大标题：`Life off-screen` 或 `Content & channels`
- 2x2 平台卡片：
  - GitHub
  - B 站
  - 小红书
  - 公众号
- GitHub 和 B 站使用可点击卡片。
- 小红书和公众号使用暗色 disabled 卡片。

移动端：

- 平台卡片单列。
- 内容方向用可换行标签展示。

### 交互效果

- GitHub / B 站 hover 时卡片上浮，外链箭头出现。
- 暂无平台 hover 时只显示“待补充”，不跳转。
- 平台卡片可以带轻微 3D 倾斜，但不能影响文字阅读。

### 内容适配

GitHub：

- 技术项目展示平台。
- 展示课程项目、Minecraft 项目、工具项目、编程学习记录。

B 站：

- 个人内容平台入口。
- 展示 UTS 留学生活、AI 专业学习、编程学习、Minecraft MOD 开发、游戏和直播相关内容。

小红书 / 公众号：

- 当前暂无。
- 保留未来内容方向。

末尾联系区延续 Hero 的 “Say hi!” 概念，做成强但简洁的收尾。访问者可以快速复制邮箱、手机号，或跳转 GitHub / B 站。

### UI 组件

# 三、响应式设计说明

## 桌面端

- 使用强 Hero：100vh、超大标题、三张倾斜入口卡片。
- Work 项目区使用两列卡片网格。
- Story 区使用左右图文拼贴。
- 顶部导航始终悬浮居中。

## 平板端

- Hero 标题缩小，卡片拼贴减少旋转幅度。
- Work 项目区改为两列或单列混合。
- Story 区图片拼贴移到正文下方。

## 移动端

- 顶部导航保留 Work / Story / Life，隐藏部分图标。
- Hero 标题控制在 2-3 行内。
- 倾斜卡片减少旋转，避免超出屏幕。
- 所有板块单列。
- 黑底正文行高加大，保证可读性。

# 四、实现注意事项

- 这个风格依赖大标题、黑白反差、渐变和卡片拼贴，不适合再回到普通浅色简历布局。
- 页面必须保持内容真实，不能因为视觉很酷就虚构工作经历、公司成果或商业数据。
- 项目截图不足时，使用高质量占位视觉，不要留空白大框。
- 导航的 Work / Story / Life 要能实际跳到对应内容。
- 所有链接和按钮都要有 hover、focus 和移动端可点击状态。
- 深色页面注意文本对比度，灰字不能低于可读范围。
- 动效要支持减少动效设置。
